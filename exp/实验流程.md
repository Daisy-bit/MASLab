对每个样本 qqq，执行以下步骤。

------

## Step 1：Initial Independent Generation

让 NNN 个 agent 独立回答问题。

记为：

r1(0),r2(0),…,rN(0)r_1^{(0)}, r_2^{(0)}, \ldots, r_N^{(0)}r1(0),r2(0),…,rN(0)

每个 agent 使用相同 base model和不同的角色。

需要保存：

- 每个 agent 的完整回答；
- 抽取出的最终答案；
- 该答案是否正确；
- token 数。

------

## Step 2：Initial Voting

从每个初始回答中抽取答案：

xi(0)=ET(ri(0))x_i^{(0)} = \mathcal E_T(r_i^{(0)})xi(0)=ET(ri(0))

其中 ET\mathcal E_TET 是任务相关 answer extractor。

对抽取后的答案做 多数投票：

V(0)=Voting⁡(x1(0),…,xN(0))V^{(0)} = \operatorname{Plurality}(x_1^{(0)},\ldots,x_N^{(0)})V(0)=Plurality(x1(0),…,xN(0))

记录：

- initial single-agent average accuracy；
- initial vote accuracy；
- initial oracle coverage。

## **Step 4:  样本分桶规则**

设 gold answer 为 yyy。

初始第 iii 个 agent 是否正确：

zi(0)=1[xi(0)=y]z_i^{(0)} = \mathbf 1[x_i^{(0)}=y]zi(0)=1[xi(0)=y]

初始投票是否正确：

Zvote(0)=1[V(0)=y]Z_{\text{vote}}^{(0)} = \mathbf 1[V^{(0)}=y]Zvote(0)=1[V(0)=y]

初始 oracle coverage：

C(0)=1[∃i, xi(0)=y]C^{(0)} = \mathbf 1[\exists i,\ x_i^{(0)}=y]C(0)=1[∃i, xi(0)=y]

根据这两个量，把样本分成三类。

------

## Type I：Already Solved

V(0)=yV^{(0)}=yV(0)=y

也就是初始投票已经正确。

这类样本上，debate 的目标不是提升，而是**不要伤害**。

如果固定多轮 debate 之后变错了，就说明存在 over-debate：

V(0)=y,V(L)≠yV^{(0)}=y,\quad V^{(L)}\ne yV(0)=y,V(L)=y

这正好支持你的 early stopping。

------

## Type II：Recoverable

V(0)≠y,C(0)=1V^{(0)}\ne y,\quad C^{(0)}=1V(0)=y,C(0)=1

也就是初始投票错了，但至少有一个 agent 是对的。

这类样本是 MAD 最有价值的地方，因为正确答案已经存在，只是没有被选中。

你的方法真正应该在这里提升：

V(L)=yV^{(L)}=yV(L)=y

这就是正确信号放大。

------

## Type III：Unrecoverable

C(0)=0C^{(0)}=0C(0)=0

所有 agent 初始都错。

这类样本上，普通 MAD 很难凭空创造正确答案。你不需要强行承诺提升，只需要诚实说明：

> 我们的方法主要针对 correct signal already exists but is under-selected 的情况。

这反而显得更严谨。

## 6.2 Initial Vote Accuracy

Accvote(0)=1M∑q1[Vq(0)=yq]Acc_{\text{vote}}^{(0)} = \frac{1}{M} \sum_q \mathbf 1[V_q^{(0)}=y_q]Accvote(0)=M1q∑1[Vq(0)=yq]

## 6.3 Final Debate Accuracy

vanilla MAD 最终投票正确率：

Accvote(L)=1M∑q1[Vq(L)=yq]Acc_{\text{vote}}^{(L)} = \frac{1}{M} \sum_q \mathbf 1[V_q^{(L)}=y_q]Accvote(L)=M1q∑1[Vq(L)=yq]

------

## 6.4 Oracle Coverage

初始回答池中至少有一个 agent 正确的比例：

Coverage(0)=1M∑q1[∃i, xq,i(0)=yq]Coverage^{(0)} = \frac{1}{M} \sum_q \mathbf 1[\exists i,\ x_{q,i}^{(0)}=y_q]Coverage(0)=M1q∑1[∃i, xq,i(0)=yq]

这个指标非常重要。

如果：

Coverage(0)≫Accvote(0)Coverage^{(0)} \gg Acc_{\text{vote}}^{(0)}Coverage(0)≫Accvote(0)

说明存在大量 recoverable cases，即正确答案已经在池子里，但没有被初始投票选出来。

------

## 6.5 Recoverable Ratio

recoverable 样本占比：

Rrecoverable=1M∑q1[Vq(0)≠yq, Cq(0)=1]R_{\text{recoverable}} = \frac{1}{M} \sum_q \mathbf 1[V_q^{(0)}\ne y_q,\ C_q^{(0)}=1]Rrecoverable=M1q∑1[Vq(0)=yq, Cq(0)=1]

这表示 MAD 潜在可利用空间有多大。

------

## 6.6 Recovery Rate

在 recoverable 样本中，debate 成功恢复正确答案的比例：

Raterecover=P(V(L)=y∣V(0)≠y, C(0)=1)Rate_{\text{recover}} = P(V^{(L)}=y \mid V^{(0)}\ne y,\ C^{(0)}=1)Raterecover=P(V(L)=y∣V(0)=y, C(0)=1)

也可以写成：

Raterecover=#{q:Vq(0)≠yq, Cq(0)=1, Vq(L)=yq}#{q:Vq(0)≠yq, Cq(0)=1}Rate_{\text{recover}} = \frac{ \#\{q: V_q^{(0)}\ne y_q,\ C_q^{(0)}=1,\ V_q^{(L)}=y_q\} }{ \#\{q: V_q^{(0)}\ne y_q,\ C_q^{(0)}=1\} }Raterecover=#{q:Vq(0)=yq, Cq(0)=1}#{q:Vq(0)=yq, Cq(0)=1, Vq(L)=yq}

这是衡量 vanilla MAD 是否能放大 correct minority signal 的核心指标。

------

## 6.7 Harmful Flip Rate

在 already-solved 样本中，debate 把正确初始投票变错的比例：

Rateflip=P(V(L)≠y∣V(0)=y)Rate_{\text{flip}} = P(V^{(L)}\ne y \mid V^{(0)}=y)Rateflip=P(V(L)=y∣V(0)=y)

即：

Rateflip=#{q:Vq(0)=yq, Vq(L)≠yq}#{q:Vq(0)=yq}Rate_{\text{flip}} = \frac{ \#\{q: V_q^{(0)}=y_q,\ V_q^{(L)}\ne y_q\} }{ \#\{q: V_q^{(0)}=y_q\} }Rateflip=#{q:Vq(0)=yq}#{q:Vq(0)=yq, Vq(L)=yq}

这是衡量 over-debate 的核心指标。

------

## 6.8 Unrecoverable Success Rate

在 unrecoverable 样本中，debate 最终答对的比例：

Rateunrec=P(V(L)=y∣C(0)=0)Rate_{\text{unrec}} = P(V^{(L)}=y \mid C^{(0)}=0)Rateunrec=P(V(L)=y∣C(0)=0)

即：

Rateunrec=#{q:Cq(0)=0, Vq(L)=yq}#{q:Cq(0)=0}Rate_{\text{unrec}} = \frac{ \#\{q: C_q^{(0)}=0,\ V_q^{(L)}=y_q\} }{ \#\{q: C_q^{(0)}=0\} }Rateunrec=#{q:Cq(0)=0}#{q:Cq(0)=0, Vq(L)=yq}

这个指标用于判断 debate 是否真的能“凭空创造正确答案”。

# 我的的方法也应该对应这两个机制

新的逻辑可以是：

## 机制一：Signal availability

问题：

> 当前是否已经有足够共识，或者是否存在值得辩论的分歧？

对应你的谱分歧指标：

D(r)=tr⁡(HSH)N−1D(\mathbf r)=\frac{\operatorname{tr}(HSH)}{N-1}D(r)=N−1tr(HSH)

作用：

- 低分歧：直接停止，避免 over-debate；
- 中等分歧：进入 controlled debate；
- 极高分歧：谨慎处理，避免全连接互相污染。

------

## 机制二：Signal amplification

问题：

> 如果要 debate，谁应该影响谁？

对应你的 spectral contribution：

cic_ici

但建议不要说它是“可靠性”，而说它是：

> runtime geometric contribution

然后路由：

wj→i=max⁡(0,Sij)max⁡(0,cj−ci)w_{j\rightarrow i} = \max(0,S_{ij})\max(0,c_j-c_i)wj→i=max(0,Sij)max(0,cj−ci)

作用：

> 让高 contribution 的回答影响低 contribution 的回答，从而减少无控制全连接带来的错误放大。
