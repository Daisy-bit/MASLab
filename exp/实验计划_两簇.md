# 实验计划：两个独立基础算法上的 SCC 模块叠加实验

不再把 `mad_vote` 与 `soo_centered_v3` 拉到同一张表里横比。改为在每个基础算法各自的初始 prompt / 拓扑 / 终止条件下，单独叠加我们提出的两个核心模块，做 within-base 对比。论文第 3 节的 "Already-flip / Recoverable Recovery / Unrecoverable Success / Coverage Drop" 四元诊断框架在两个簇上各跑一遍。

**全局约束**：

- 所有实验 `diversity_p = 0`（关闭随机扰动）。
- 簇 B `max_rounds = 3`（与簇 A 的 3 轮对齐）。
- macro 平均（每个 dataset 等权）贯穿所有 Avg. 行。

---

## 簇 A：以 mad_vote 为基础

**Base**：`methods/mad_vote`，5 agents, fully-connected, fixed 3 rounds, plurality vote, 5 个推理 persona prompt 不变。

**叠加三个 SCC 模块**：

- **谱共识早停 (Spectral Consensus Triggering)**：每轮算 round-0 + 中途的 `tr(S_c)`（来自 `methods/soo_centered_v2`），以及 answer-level 多数计票（≥3 同意即停）。命中即返回当轮 plurality 结果，跳过后续轮。
- **DAG 编排 (Contribution-guided Routing)**：把 round t→t+1 的 fully-connected peer broadcast 改成基于贡献分（PC1 from spectral）的 top-k=2 稀疏 DAG，**`diversity_p = 0`**（无随机扰动）。
- **加权聚合 (Contribution-weighted Aggregation)**：round-0 与每轮的 plurality 计票从"每个 agent 1 票"改成"每个 agent 按其 PC1 softmax 贡献分加权"。这是 SCC 论文与 vanilla MAD 的核心差异点之一，必须独立消融。

**变体（5 行 × 3 模型 × 5 数据集）**：

| Variant | Triggering | Routing | Aggregation | 来源 |
|---|---|---|---|---|
| A0. mad_vote (vanilla) | fixed 3 rounds | full mesh | plurality (等权) | 已有 `results_diagnostic/run_2026050*` |
| A1. mad_vote + Triggering | spectral+vote stop | full mesh | plurality (等权) | 新建变体 |
| A2. mad_vote + Routing | fixed 3 rounds | top-k DAG (`diversity_p=0`) | plurality (等权) | 新建变体 |
| A3. mad_vote + Aggregation | fixed 3 rounds | full mesh | weighted (贡献加权) | 新建变体 |
| A4. mad_vote + All | spectral+vote stop | top-k DAG (`diversity_p=0`) | weighted (贡献加权) | 新建变体 |

**实现策略**：单独实现在 `methods/mad_vote_scc/`（继承自 `MAD_Vote_Main`），三个 config 开关 `enable_triggering / enable_routing / enable_weighted_aggregation`。Spectral / 贡献分计算自包含（不再继承 SOO/SelfOrg），与 `soo_centered_v2` 的 `_double_center / _approx_shapley` 同公式但独立实现。

**输出位置**：`results_madvote_scc/run_<TS>/<variant>/<model>/_tables/table{1..6}_*.csv`

## 簇 B：以 soo_centered_v3 为基础

**Base**：`methods/soo_centered_v3` config_diag（已有 `emit_diagnostic: true`），5 agents, top-k=2 DAG, **`max_rounds = 3`**, 角色 prompt + few-shot + 答案早停，**`diversity_p = 0`**。

**叠加 / 拆解的两个观察轴**：

- **初始多数 vote 强度**：round-0 的 5 agent 答案抽取 → plurality vote。这是 SCC 提前停的入口。
- **每轮 rounds 的演化**：把 round-0/round-1/round-2/round-3 各自的 vote 答案分别保存，看每轮的 Acc / 各桶 transition。强制 emit 每轮 vote 字段。

**变体（4 行 × 3 模型 × 5 数据集，需要重跑因为 `max_rounds` 由 2 改为 3、`diversity_p` 由 0.2 改为 0）**：

| Variant | Triggering | Routing | Aggregation |
|---|---|---|---|
| B0. SCC-full | adaptive | spectral DAG (`diversity_p=0`) | conservative |
| B1. w/o Triggering | fixed 3 rounds | spectral DAG (`diversity_p=0`) | conservative |
| B2. w/o Routing | adaptive | full mesh | conservative |
| B3. w/o Aggregation | adaptive | spectral DAG (`diversity_p=0`) | plurality |

**输出位置**：`results_scc_internal/run_<TS>/<variant>/<model>/_tables/`

分析层增量：

1. 给 `analyze_diagnostic.py` 扩展 `table5_roundwise_accuracy`：**per-round 四桶 transition**（哪些 already→flip、哪些 recoverable→recover 发生在 round 1 / 2 / 3）。
2. 增 `table7_round_exit_distribution.csv`：每个 variant 在 round-0 / round-1 / round-2 / round-3 退出的样本比例，验证 Triggering 是否有效。
